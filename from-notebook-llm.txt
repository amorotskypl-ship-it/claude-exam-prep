2. The Five Core Architectural Mental Models
Architectural success requires a shift from "chat-based" interaction to "systems-based" thinking. These models represent the philosophical foundation of the Intelligence Layer.
1. Programmatic Enforcement vs. Prompt-Based Guidance
In production, prompt-based instructions are probabilistic. For compliance-critical or financially sensitive workflows (e.g., identity verification before a refund), architects must implement programmatic enforcement. This involves using SDK hooks or tool-call interceptors to create "hard" deterministic gates. Relying on an LLM to "usually" follow a policy is an architectural failure; guardrails must be enforced at the system level.
2. Tool Descriptions as Primary Routing
In agentic architectures, the description of a tool—not its name or the system prompt—is the critical mechanism for model routing. Descriptions override system prompt keywords. Effective design requires definitions that define explicit boundaries, input formats, edge cases, and example queries to avoid "hallucinated" calls or selection ambiguity.
3. Context Isolation (Subagents Do Not Inherit)
A common architectural trap is assuming subagents share memory with the parent coordinator. In the Claude Agent SDK, subagents do not automatically inherit conversation history. Every data point required for a subtask must be explicitly passed in the subagent's prompt. This isolation prevents context bloat but necessitates explicit information passing by the architect.
4. The "Lost in the Middle" Effect
Long-context models suffer from a physical design constraint where performance peaks at the beginning and end of a context window but degrades in the center. Architects must design inputs to place key instructions and summaries at the top, use explicit section headers, and aggressively trim verbose tool outputs before they accumulate in the middle of the window.
5. Batch API vs. Real-Time Latency
Choosing between APIs is a tradeoff of latency and SLA, not just cost. The Message Batches API offers a 50% cost reduction but is constrained by a 24-hour processing window with no guaranteed SLA and no multi-turn tool support. Consequently, real-time user-facing apps must use the standard Messages API, while background audits and data extractions are the only appropriate batch candidates.

3. Domain 1: Agentic Architecture & Orchestration (27%)
The Agentic Loop
The lifecycle is managed via stop_reason values:
tool_use: The model pauses, requiring the system to execute a tool and return the result to continue.
end_turn: The model has completed reasoning and terminates the loop.
Anti-Pattern: Parsing natural language to detect the loop's end instead of using stop_reason.
Multi-Agent Patterns: Hub-and-Spoke
The Coordinator (Hub) decomposes tasks and delegates to Subagents (Spokes). Coordinator Responsibilities:
Decompose high-level goals into discrete subtasks.
Delegate tasks using the Task or Agent tool.
Aggregate findings and resolve conflicts (Multi-source synthesis).
Route errors and manage autonomous recovery logic.
Deterministic Guardrails (SDK Hooks)
Hooks allow architects to inject deterministic logic into the agent loop.
Note: Programmatic hooks (passed to query()) are scoped only to the main session, while filesystem hooks (settings.json) fire for the main agent and all subagents.

4. Domain 2: Tool Design & MCP Integration (18%)
MCP Primitives
The Model Context Protocol (MCP) modularizes the agent's "senses":
Tools: Executable functions that change state or query dynamic data.
Resources: Static/semi-static data (e.g., database schemas, log files).
Prompts: Templates that guide the model's interaction with specific datasets.
Structured Error Handling
Tools must return structured metadata. The isRetryable flag is the core mechanism for autonomous recovery:
Transient Errors: (Timeouts/Unavailability) isRetryable: true.
Validation/Business/Permission Errors: (Invalid input/Policy gap) isRetryable: false.
Anti-Pattern: Silently returning empty results as a success, which prevents coordinator recovery.
Configuration Scoping
Project-level (.mcp.json): Shared via version control; standardizes tools for the team.
User-level (~/.claude.json): Personal configurations and OAuth sessions; not shared.
Tool Provisioning
Architects should target 4–5 focused tools per agent. Selection reliability degrades significantly when agents are granted universal access to 10+ tools.

5. Domain 3: Claude Code Configuration & Workflows (20%)
CLAUDE.md Hierarchy
CLAUDE.md defines architectural decisions and coding standards.
Specialized Workflows & Flags
--worktree: Uses sparsePaths and symlinkDirectories to reduce disk usage and startup time in large monorepos.
--dangerously-skip-permissions: Used for non-interactive CI/CD to prevent pipeline hangs (requires isolation).
context: fork: Skill frontmatter that spawns an isolated subagent, preventing session pollution.

6. Domain 4: Prompt Engineering & Structured Output (20%)
Structured Output via tool_use
JSON schemas are the most reliable way to enforce output contracts.
Nullable Fields: Prevents the model from fabricating information absent from the source.
"Unclear" Enums: Explicitly allow the model to admit ambiguity.
"Other" + Detail String: Allows for extensible categorization.
Validation & Retry Loops
The "Self-Correction" pattern: If validation fails, feed the original prompt, the failed code, and the specific validation error log back to the model for a correction turn.
Economics & Optimization
Few-Shot Prompting: Use 2–4 targeted examples for ambiguous scenarios (e.g., distinguishing security bug severities).
Message Batches API: 50% cost savings for high-volume, single-turn tasks with a 24-hour SLA. Use custom_id for correlation.

7. Domain 5: Context Management & Reliability (15%)
Operational Reliability
Context Degradation: In long sessions, use scratchpads to record key findings and compact_boundary to manage state after summarization.
HITL Escalation: Required when (1) Explicit user request, (2) Policy gap, (3) Lack of progress after multiple retries.
Stratified Random Sampling: Used to measure ongoing error rates and detect novel patterns in high-confidence extractions.
Confidence Calibration: Calibrate model-reported confidence against labeled validation sets to set human review thresholds.
Economic Optimization
Prompt Caching: 90% reduction in read costs (0.1x rate) for repeated prefixes (codebases/documentation).
Developer Benchmark: Active developers typically spend roughly $6/day on Claude Code tokens.
Subscription Tiers: Pro (20), Max (100), Team Standard (25-30), Team Premium (150).

8. Technical Glossary: Flashcard Definitions
stop_reason: Metadata indicating why the model paused (tool_use, end_turn, max_tokens, stop_sequence).
MCP Primitives: Tools (functions), Resources (data), and Prompts (templates for specific datasets).
Prompt Caching: Mechanism for 90% cost reduction on repeated prefixes by caching the context window state.
context: fork: Skill frontmatter that prevents session pollution by spawning an isolated subagent.
isRetryable: Boolean metadata used for autonomous recovery decisions regarding transient vs. business errors.
Lost in the Middle: The degradation of attention/recall in the center of long context windows.
compact_boundary: A SystemMessage subtype used to manage window state after context compaction occurs.
